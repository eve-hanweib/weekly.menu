# 部署与多设备说明

## 部署到 GitHub（GitHub Pages）可以吗？

**可以。** 把项目部署到 GitHub Pages 后，任何人用任何设备打开同一个网址（例如 `https://你的用户名.github.io/点菜/`）都能用。

但要注意：

### 仅部署到 GitHub ≠ 同一账号在不同设备看到同一份数据

- 数据存在**每个设备自己的浏览器**（localStorage）里，**没有服务器存账号数据**。
- 所以：电脑上创建的「小明家」和选菜，**不会**自动出现在手机或别人的设备上。
- 同一账号密码在手机「登录」会进入同一个家庭，但手机本地是空的，所以看到的是空选菜/空菜单。

### 怎么让电脑和手机（或不同人）看到同一份数据？

**方式一：导出 / 导入（已加在账号页）**

1. **电脑**：登录家庭账号 → 账号设置 → **导出数据**（会复制一串数据到剪贴板）。
2. 把这段数据发给自己（微信/邮件/记事本）。
3. **手机**：用**同一账号密码**登录 → 账号设置 → **导入数据** → 粘贴 → 确认导入。
4. 手机上的选菜、菜单、采购、自定义菜品就会和电脑一致。

之后若在电脑又改了，再导出一份，在手机再导入一次即可。

**方式二：以后加真正的「云端同步」**

如果希望「同一账号在任何设备登录都自动看到同一份数据」，需要：

- 有一个**后端服务器**或**云数据库**（如 Firebase、Supabase、自建 API），把家庭数据按「家庭 ID」存到云端；
- 前端在登录/切换时从云端拉数据，修改时往云端写数据。

这需要改架构、写后端或接 BaaS，不是「只部署到 GitHub」就能实现的。

## 如何部署到 GitHub Pages

1. 在 GitHub 新建一个仓库（例如名字叫 `menu` 或 `点菜`）。
2. 把本地项目里的文件推上去（至少要有 `index.html`、`styles.css`、`app.js`、`data.js` 等）。
3. 仓库设置里打开 **Pages**：  
   **Settings → Pages → Source** 选 **main** 分支、**/ (root)**，保存。
4. 等一两分钟，访问：  
   `https://你的用户名.github.io/仓库名/`  
   例如：`https://hanwe.github.io/menu/`。

这样不同的人、不同的设备都可以打开这个链接，用各自的「家庭名称 + 密码」登录；  
数据仍在本机，要同步请用上面的**导出/导入**。
